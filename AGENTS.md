# RE:BORN — Telegram WebApp mini-game

Vanilla HTML/JS/CSS PC-builder game embedded in Telegram via `telegram-web-app.js`. Players assemble PCs from dropped components, send the build payload back to the bot via `Telegram.WebApp.sendData()`, and the bot credits real-rouble bonuses to the player's club account.

## Production vs dev — read this first

| Aspect | Where it lives |
|---|---|
| Production deploy | **GitHub Pages**: https://nik45114.github.io/reborn-game/ (origin = `github.com/nik45114/reborn-game`). The bot's `WEBAPP_URL` points here. Telegram caches WebApp HTML aggressively — `?v=` query in `index.html` + `Date.now()` cache-bust on script tags exist for this reason. |
| Dev workflow | **GitLab**: `recore/reborn-game` on reborndev.ru. Codex works here, MRs reviewed, merged to `main`. |
| GitHub sync | **Manual for now.** When `main` on GitLab is ready to ship, someone (currently Влад) pulls the changes into the GitHub origin and pushes — Pages picks it up within a minute. A CI mirror job is a planned follow-up. |
| Live leaderboard | `ranking.json` is **auto-generated by the bot on panel.recore.su** and synced into GitHub via a Windows Scheduled Task on Влад's PC every 5 min (see `../sync_ranking.py`). **Do not hand-edit `ranking.json`.** It will be overwritten. |

## Local dev

```bash
git clone https://reborndev.ru/git/recore/reborn-game.git
cd reborn-game
python -m http.server 8000
# open http://localhost:8000/ in a browser
```

The page renders fine standalone — `game.js` short-circuits on missing `window.Telegram`. Real `sendData` won't fire (you'll see `alert("Telegram.WebApp.sendData недоступен")` if you press «Собрать ПК»), but visual changes, drop animations, inventory, and the rating tab work for visual review.

For a more realistic test, use the Telegram **test environment**:
1. Open @reborntmb_bot on a phone or Desktop client
2. `/start` → bind your phone → tap «🎲 Мини-игра»
3. WebApp loads the **GitHub Pages URL**, not your local one

If you need to test server interactions, point the bot at a dev WebApp URL temporarily — but that touches `bot.py`, not this repo.

## Game contract with the bot (don't break)

When a build is assembled the WebApp calls:

```js
Telegram.WebApp.sendData(JSON.stringify({
  event: "build_assembled",
  tier: "...",       // localized tier name
  stars: 1..5,
  bonus: 25|50|100|200|350,   // CLIENT-CLAIMED — ignored by bot
  power: <int>,               // CLIENT-CLAIMED — ignored
  build: { cpu: {...}, gpu: {...}, ... }   // <-- this is what the bot trusts
}));
```

The bot has an anti-cheat path (`_validate_and_score_build` in `bot.py`) that re-derives tier and power **server-side** from `build`. It looks up each component in `cases.js`-equivalent ranges:

- `common` → 10–30 power
- `rare` → 31–60
- `epic` → 61–85
- `legendary` → 86–100

Tier thresholds: 0 ★(<150) · 1★(150) · 2★(300) · 3★(470) · 4★ — all-legendary = 5★. If you change power ranges or thresholds, update [bot.py:`_RARITY_POWER_RANGES`, `_TIER_POWER_THRESHOLDS`](https://reborndev.ru/git/recore/reborn-game/-/tree/main) too or builds will reject as cheating.

`ranking.json` is read by `game.js` on the «🏆 Рейтинг» tab:

```js
fetch("./ranking.json?ts=" + Date.now())
```

Schema:
```json
{
  "generated_at": "ISO-8601",
  "players": [{ "tg": 123, "name": "...", "builds": N, "total": 300.0, "rank": 1 }, ...]
}
```

## Push workflow

Default remote is `origin` = GitLab.

```bash
git checkout -b feature/<short-name>
# … edit, test locally …
git add -p && git commit -m "feat: …"
git push -u origin feature/<short-name>
# open MR via glab:
glab mr create -t "..." -d "..." -a
```

Don't push to `main` directly — it's the production source. Open an MR to `main`.

## Caching gotchas (real, will bite you)

- Telegram WebView caches JS/CSS aggressively. `index.html` already does `<script src="game.js?ts=<now>">`. If you add a new asset (font, image), include a `?v=N` query and bump it.
- The `app-version` badge in the bottom-right of `index.html` (`v=123`) is a manual marker — bump it when you ship a visible change so you can confirm "did my update actually reach my phone?"
- After merging to GitHub, the CDN may take ~1 min before phones see the new HTML.

## What NOT to touch (without coordination)

- `ranking.json` — automated
- `WEBAPP_URL` in the bot — moving the production URL breaks every existing user's deep links
- The `sendData` JSON schema — bot has matching parsers; coordinate breaking changes
